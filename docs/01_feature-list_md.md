# 🧭 機能一覧

---

## 前提定義（最初に決める）

| 項目     | 内容                    |
| ------ | --------------------- |
| 対象ユーザー | 会計・一般部員 |
| 提供価値   | 会計処理の時間短縮、簡略化 |
| コア体験   | 支払いの金額確認と支払い記録(会計)/支払い情報の入力(一般部員) |
| 成功指標   | 1人当たりの支払い時間を30秒以内に収めること |

---

# 機能カテゴリ設計（自由定義）

> ※カテゴリはプロダクトに応じて自由に再設計する
> 例：アカウント / 管理 / コア機能 / 可視化 / 連携 / 運用 など

---

## 【カテゴリA：認証】

| #   | 機能名 | 概要 | ユーザー価値 | 優先度   | 備考 |
| --- | --- | -- | ------ | ----- | -- |
| A-1 | 画面単位のアクセス制御 | イベント作成や支払い記録は会計のみがアクセス可能にする | 支払いを改竄されないセキュリティを保証する | 🔴 P0 |    |
| A-2 | OAuthログイン | Discordアカウントを利用した認証 | 一般部員にも認証を提供し、場合によっては支払い登録の修正機能なども実現可能にする | 🔴 P0 |    |
| A-3 | ログイン状態の保持 | クライアントにトークンを保持して一定期間認証操作をスキップ可能にする | 認証操作の回数・頻度が減る | 🔴 P0 |    |

---

## 【カテゴリB：コア機能】

| #   | 機能名 | 概要 | ユーザー価値 | 優先度 | 備考 |
| --- | --- | -- | ------ | ----- | -- |
| B-1 | 支払い記録 | 支払い金額の確認と支払い時刻の記録をする | シンプルなUIで素早く支払い記録ができる | 🔴 P0 |    |
| B-2 | 支払い登録 | 自身が支払う対象の商品を選択・登録する | 分かりやすいUIで統一的に支払い登録ができる | 🔴 P0 |    |
| B-3 | イベント作成 | 支払いが生じるイベントの名前と選択可能な商品・金額を登録する | 最低限の入力だけで支払い受付の準備を整えられる | 🟡 P1 |    |
| B-4 | 支払い情報検索 | イベントごとにユーザーの条件で検索して登録した支払い情報を検索する | 支払い情報を簡単な条件で検索できる | 🟡 P1 |
| B-5 | 支払いユーザーリスト閲覧 | イベントごとに支払いユーザーの一覧を閲覧する | 支払い記録をするユーザーを選択できる | 🔴 P0 |
| B-6 | イベントリスト閲覧 | 支払い受付中のイベントの一覧を閲覧する | 支払い登録などの操作を行うイベントを選択できる | 🔴 P0 |

# 優先順位の定義基準（共通ルール）

| 優先度   | 判断基準              |
| ----- | ----------------- |
| 🔴 P0 | これがないとプロダクトが成立しない |
| 🟡 P1 | 体験を大きく向上させる       |
| 🟢 P2 | 運用効率・継続率を改善する     |
| ⚪ P3  | 将来の拡張性・差別化        |
