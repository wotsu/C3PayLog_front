# 技術スタック

---

## 前提定義（最初に決める）

| 項目        | 内容                                 |
| --------- | ---------------------------------- |
| プロダクト種別   | SaaS |
| 想定ユーザー規模  | 100人程度 |
| 可用性目標     | 99.9% |
| セキュリティ要件  | 個人情報レベル |
| パフォーマンス要件 | レスポンスタイム2秒以内 |
| チーム体制     | 2人 / 専任なし |
| リリース頻度    | 月次 |
| 予算制約      | ほぼ0(VPSの費用などを除く) |
| 既存資産      | VPS、ドメイン、CDN、Discord OAuth技術 |

# 技術スタック構成

## 🖥 フロントエンド

| 項目      | 採用技術 | 採用理由 | 評価観点                     | 備考 |
| ------- | ---- | ---- | ------------------------ | -- |
| 言語      | TypeScript | 既存の部内サービスと統一して学習コスト低減 | 学習コスト |    |
| フレームワーク | なし | 既存サービスと統一、小規模なので不要 | 学習・保守コスト |    |
